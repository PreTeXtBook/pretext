# `examples/numbering` — a numbering stress test

`main.ptx` is a synthetic PreTeXt document that exercises the numbering system
as exhaustively as we could manage: every numbered family (divisions; blocks
and their `distinct` figure / project / exercise variants; equations;
footnotes; figures, tables, and listings) placed in every scope that affects a
number — introduction/conclusion "zero-slot" pooling, nested subdivisions,
specialized divisions (`exercises`, `references`, `glossary`, …), front and
back matter, structural parts on and off. It is built repeatedly across the
range of `numbering/*/@level` settings and the outputs are diffed.

## Heads-up: machine-made, and machine-oriented

This document was **generated by Claude (Anthropic's AI assistant)** over a
long, iterative run of actual before/after numbering testing (the
assembly-time `@serial` rework for equations and footnotes). It grew to fit the
corner cases that testing kept turning up.

It is **not** an example of good PreTeXt authoring, and is not really intended
for human eyes:

- The prose is deliberately throwaway ("First theorem", `a = b`, "A
  placeholder") so that numbers are easy to predict, grep, and diff.
- It crowds together constructs no real document would — every block type in
  one section, numbered objects inside introductions and conclusions, and so
  on — purely to cover edge cases.
- Its purpose is **automated**: regression diffs, level sweeps, and
  scope/pooling checks, not reading or pedagogy.

Treat it as a numbering test fixture, not as sample authoring.

## A no-parts variant, on demand

`main.ptx` uses structural `<part>`s. To test the part-free structure —
chapters directly under `<book>`, no `<part>` at all — derive a throwaway copy
by deleting the two `<part>` wrappers (their start/end tags and the part
`<title>`s) so the chapters become children of `<book>`. It regenerates
trivially, so it is not kept in the repository. (Toggling `part-structure`
between `structural` and `decorative` in a publication file already exercises
parts on and off; this variant is only for the no-`<part>` structure.)
